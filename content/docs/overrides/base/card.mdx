The Card uses the same structure as the shadcn Card. Install when your style is
`base-nova`, `base-vega`, etc.

## Install

Configure your registry in `components.json`:

```json
{
  "registries": {
    "@uxio": "https://ui.uxio.dev/r/styles/{style}/{name}.json"
  }
}
```

Then run:

```bash
npx shadcn@latest add @uxio/card
```

The correct variant (base or radix) is resolved automatically from your `components.json` style.

## Extended Features

This override extends the original shadcn Card with a variant prop (default/destructive) using class variance authority.

- [Default Variant](#default-variant)
- [Destructive Variant](#destructive-variant)
- [Style Changes](#style-changes)

### Style Changes

The following changes apply to the default style (no examples needed):

- **`data-variant`**: The card has an extra attribute `data-variant={variant}` for styling hooks.
- **Destructive ring**: When `variant="destructive"`, the card has `ring-destructive/30 dark:ring-destructive/60` applied.
- **Footer styling**: The footer has added classes for destructive variant support: `border-t bg-muted group-data-[variant=destructive]/card:[.border-t]:border-destructive/30 group-data-[variant=destructive]/card:[.border-t]:bg-destructive/5`.
