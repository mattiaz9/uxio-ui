## Overview

Two-column layout on a grid with outer inset columns (`--container-inset` from **Tailwind Container Extra**). Each side is a `SplitContentItem` that can align to the **container** or **screen** edge. Use the **responsive** variant to stack on small viewports, or **fixed** to keep two columns at all widths.

## Install

Add **Tailwind Container Extra** if you have not already (required for `--container-inset`):

```bash
npx shadcn@latest add @uxio/tw-container-extended
```

Then:

```bash
npx shadcn@latest add @uxio/split-content
```

## Usage

```tsx
import { SplitContent, SplitContentItem } from "@/components/ui/split-content"

export function HeroSplit() {
  return (
    <SplitContent className="divide-y divide-border md:divide-x md:divide-y-0">
      <SplitContentItem anchor="container">…</SplitContentItem>
      <SplitContentItem anchor="screen">…</SplitContentItem>
    </SplitContent>
  )
}
```

You must pass **exactly two** `SplitContentItem` children (no wrappers that break direct children). Other node types (e.g. whitespace-only text) are ignored for that count.

## Props

### `SplitContent`

| Prop        | Type                                   | Default        | Description                                                                 |
| ----------- | -------------------------------------- | -------------- | --------------------------------------------------------------------------- |
| `variant`   | `"responsive"` \| `"fixed"`            | `"responsive"` | **responsive**: stack on small screens, two columns from `md`. **fixed**: always two columns. |
| `ratio`     | `"1:1"` \| `"1:2"` \| `"2:1"`          | `"1:1"`        | Relative column widths at the multi-column breakpoint.                      |
| `className` | `string`                               | —              | Merged with layout utilities on the root `<section>`.                       |
| …           | `React.ComponentProps<"section">`      | —              | Standard section props (`id`, `aria-*`, etc.). `children` must be two items. |

Root element exposes `data-slot="split-content"`, `data-variant`, and `data-ratio`.

### `SplitContentItem`

| Prop        | Type                            | Description                                                                 |
| ----------- | ------------------------------- | --------------------------------------------------------------------------- |
| `anchor`    | `"container"` \| `"screen"`     | **container**: align to the centered content column. **screen**: extend into the inset gutters toward the viewport edge. |
| `className` | `string`                        | Merged with column span utilities.                                          |
| …           | `React.ComponentProps<"div">`   | Standard div props.                                                         |

`data-slot="split-content-item"` and `data-anchor` are set for styling hooks.

### Screen anchor: full-bleed background, container-aligned text

With **`anchor="screen"`**, the grid cell reaches into the outer inset—handy for **full-bleed photos, gradients, or video** in that column while the main content grid stays predictable.

Backgrounds usually belong on the item (or a filling child) so they span the whole cell. Wrap **text** in an inner element and add **`pl-container-inset`** on the **first** column or **`pr-container-inset`** on the **second** so copy lines up with the same horizontal start/end as **`anchor="container"`** columns. Both utilities are included when you install **Tailwind Container Extra** (`@uxio/tw-container-extended`).
