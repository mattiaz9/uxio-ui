  Numeric input built on Input Group with inline step buttons, filtered text entry, and separate
  callbacks for visible text and committed numeric value.
---

## Overview

`InputNumber` composes **Input Group** with a visible text input and two suffix step buttons. It
accepts either **string** or **number** values, filters non-numeric characters as the user types,
and keeps `onChange` separate from `onValueChange`.

- Use `onChange` for the visible input text.
- Use `onValueChange` for the committed numeric value.
- Blur, `Enter`, `ArrowUp`, `ArrowDown`, and the suffix buttons all perform a commit.
- Wheel changes are disabled so scrolling over the field does not change the value.

## Custom props

| Prop            | Type                                   | Description                                                                 |
| --------------- | -------------------------------------- | --------------------------------------------------------------------------- |
| `value`         | `string \| number`                     | Controlled value. Strings control the visible text; numbers control the numeric state. |
| `defaultValue`  | `string \| number`                     | Uncontrolled initial value.                                                 |
| `onChange`      | `ChangeEventHandler<HTMLInputElement>` | Fires with the visible string while typing and again on commit.             |
| `onValueChange` | `(value: number \| null) => void`      | Fires only when the committed numeric value changes.                        |
| `min`           | `number`                               | Optional minimum clamp. Ignored if `min > max`.                             |
| `max`           | `number`                               | Optional maximum clamp. Ignored if `min > max`.                             |
| `step`          | `number`                               | Step size for buttons and arrow keys. Invalid or non-positive values fall back to `1`. |

All remaining props come from the underlying `Input` primitive and are spread onto the visible
input field.

## Install

Configure your registry in `components.json`:

```json
{
  "registries": {
    "@uxio": "https://ui.uxio.dev/r/styles/{style}/{name}.json"
  }
}
```

Then run:

```bash
npx shadcn@latest add @uxio/input-number
```

This pulls `input-group` as a registry dependency. The correct variant follows your
`components.json` style.

## Usage

### String-controlled

```tsx
import * as React from "react"

import { InputNumber } from "@/components/ui/input-number"

export function QuantityField() {
  const [text, setText] = React.useState("")
  const [committed, setCommitted] = React.useState<number | null>(null)

  return (
    <InputNumber
      value={text}
      min={0}
      max={10}
      step={0.5}
      onChange={(event) => setText(event.target.value)}
      onValueChange={setCommitted}
    />
  )
}
```

### Number-controlled

```tsx
import * as React from "react"

import { InputNumber } from "@/components/ui/input-number"

export function PriceField() {
  const [value, setValue] = React.useState(12.5)

  return (
    <InputNumber
      value={value}
      step={0.25}
      onChange={(event) => {
        console.log("visible text", event.target.value)
      }}
      onValueChange={(next) => {
        if (next !== null) setValue(next)
      }}
    />
  )
}
```
